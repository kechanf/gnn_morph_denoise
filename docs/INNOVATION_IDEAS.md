# 如何增强实验创新性（2026 视角）

当前使用的 GCN/GAT/SAGE/GIN 属于 2017–2019 年的经典 GNN，在 2026 年审稿中容易被认为“方法偏旧”。**创新不必堆在网络上**，可以从数据、训练、评估、几何与可解释性等多方面提升，下面按优先级和可落地性整理。

---

## 一、现状：为什么说“过于简单”

- **结构**：2–4 层消息传递 + 全连接分类头，无位置/结构编码，无显式几何。
- **消息**：仅用节点特征 + 邻接，未用边特征（距离、角度、半径差等）。
- **训练**：监督 CE + 准确率，无对比/自监督、无不确定性。
- **评估**：整体节点准确率，未按噪声类型、图大小、长程依赖等细分。

这些在 2026 年都属于“基线级”设定，适合作为对照，但单独作为贡献会显得薄弱。

---

## 二、网络侧增强（适度即可，不必追求最前沿）

### 2.1 低成本、高收益

| 方向 | 做法 | 创新点简述 |
|------|------|------------|
| **位置/结构编码** | 在节点特征中拼接 Laplacian 特征向量（前 k 维）或 RWPE | 打破同构图的对称性，增强对结构的表达能力，与 “Weisfeiler–Lehman” 型 GNN 形成互补。 |
| **Jumping Knowledge (JK)** | 把每层的节点表示 concat 或 LSTM 聚合后再分类 | 缓解过平滑，同时利用多跳信息，适合树上的长程依赖。 |
| **Initial / Residual 连接** | 每层 `h' = h + MLP(AGG(h))` 或首层残差 | 减轻深层过平滑，稳定训练。 |

以上都可以在现有 `models/build.py` 上做**对照实验**（有/无 PE、有/无 JK），写进论文即“方法改进 + 消融”。

### 2.2 几何感知消息传递（与任务强相关）

你的数据有 3D 坐标、半径、角度，但当前 **边只用于拓扑**，未把几何信息放进消息里：

- **边特征**：`edge_attr = [dist_ij, angle_ij, radius_i, radius_j]`（或归一化版本），在 GCN/GAT 中用 `edge_attr` 调制消息权重。
- **或使用 PNA、EGNN、SchNet 等**：在消息里显式用距离/方向，更适合“形态学几何”。

**创新点**：在神经元形态图上证明“几何边特征”比“仅拓扑”更有效，并做消融（仅拓扑 vs +距离 vs +角度）。

### 2.3 若要做“更现代”的架构

- **Graph Transformer / GraphGPS**：用全局注意力 + 局部 MP，或 Laplacian PE + Transformer。适合作为“强基线”或一个对比模型，不必作为主贡献。
- **图级/子图级辅助**：若存在“整图是目标/干扰”的标签，可加图级分类头做多任务，增强节点表示。

建议：**主贡献放在数据/任务/评估上，网络以 2.1 + 2.2 为主**，再选一个 Graph Transformer 做对比即可。

## 三、非网络创新（更易出亮点）

### 3.1 数据与增强

- **图数据增强**：训练时随机 DropEdge、Feature Mask、子图采样，或对坐标/半径加小噪声，验证鲁棒性与泛化。
- **按策略拆分评估**：在合成数据上按 `full_tree` / `local_spur` / `break_fragment` 等分别报准确率，分析“模型在哪类噪声上弱”，并针对性加数据或损失。
- **课程/难例挖掘**：先训简单样本（如仅 full_tree），再逐步加入 break_fragment 等难例；或对高损失样本过采样。

**创新点**：系统分析“合成噪声类型 vs 模型性能”，并给出数据或训练上的改进策略。

### 3.2 训练目标

- **对比学习**：同一图中“目标节点”拉近、“干扰节点”推远，或图级对比（不同图分开），再用现有分类头做下游。可引用 GraphCL/BGRL 等，适配为“节点级 + 你的任务”。
- **辅助任务**：同时预测半径、节点类型、到根距离等（多任务），或做边/节点重构，提升表示质量。
- **类别不平衡**：若目标/干扰比例不均，用 Focal Loss 或加权 CE，并报告 per-class 指标。

**创新点**：在形态学节点分类任务上，证明“对比或多任务”相对纯 CE 的收益，并分析为何有效。

### 3.3 评估与泛化

- **按噪声类型报告**：不仅 overall accuracy，还按合成策略（或按断点/碎片比例）分层报告，画表格或曲线。
- **跨数据集**：在 A 池合成训练、在 B 池或真实标注数据上测试，验证泛化。
- **不确定性**：简单做法如 MC Dropout 或小集成，给出每个节点的置信度或校准曲线，便于下游追踪决策。

**创新点**：建立“形态学节点分类”的细粒度评估协议（按噪声类型 + 泛化 + 不确定性），可单独作为贡献。

### 3.4 几何与领域知识

- **显式几何特征**：除了现有 6 维，可加“到根路径长度”“分支阶数”“局部曲率”等，作为节点或边特征。
- **树结构先验**：例如对“根→叶”路径上的节点做序列建模（LSTM/Transformer），或对兄弟分支做对称约束，与 GNN 结合。

**创新点**：把形态学先验（树、路径、分支）形式化并融入模型或特征，突出领域特色。

### 3.5 可解释性与应用

- **注意力/梯度**：用 GAT 注意力或梯度对节点/边做重要性分析，看模型更依赖哪些节点或特征（如半径、类型）。
- **失败案例分析**：挑出预测错的节点，与 break_fragment、距离根远近等做定性分析，指导数据合成或特征设计。
- **下游集成**：与现有追踪流程结合（如用预测掩码剪枝、引导人工校对），报告效率或质量提升。

**创新点**：从“可解释性”或“在真实流程中的价值”角度写，而不只是准确率数字。

## 四、推荐组合（最小改动拿得出手）

1. **必做（低成本）**  
   - 在现有 GNN 上增加 **Laplacian PE 或 RWPE**，做一组有/无 PE 的对照。  
   - 在评估上 **按合成策略（或噪声类型）分层报结果**，并写一段分析。

2. **选做（二选一或都做）**  
   - **几何边特征**：在消息传递中加入距离/角度等 edge_attr，做消融。  
   - **对比或多任务**：加一个对比损失或辅助预测任务，与纯 CE 对比。

3. **加分项**  
   - 一个 **Graph Transformer / GraphGPS** 基线；  
   - **不确定性**（MC Dropout）或简单集成；  
   - 简短 **可解释性/失败案例** 小节。

这样可以在不大改代码的前提下，把“经典 GNN + 合成数据”升级为“**带位置/几何与细粒度评估的形态学节点分类**”，创新点清晰且可复现。若你指定一个方向（例如先做 PE 或先做按策略评估），我可以按你当前代码结构给出具体改法（含 config/experiment 设计）。
